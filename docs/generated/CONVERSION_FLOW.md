# Generated Conversion Flow

Generated by `scripts/docs/generate_repo_docs.py`. Do not edit by hand.

## Runtime Flow

1. User action enters SwiftUI views or Quick Action handoff.
2. `ConversionQueue` owns job state, entitlement-gated enqueue, retry, cancellation, progress, and result/error state.
3. `ConversionRunner` copies the source into an app-owned workspace and chooses the concrete extraction path.
4. Apple-native paths are attempted where valid: PDFKit, Vision, Speech, ImageIO, AVFoundation, plus native Office/HTML/text/EPUB engines.
5. The AI path runs native Granite-Docling (mlx-swift) in-process. Conversion is native-only — there is no Python runtime or helper process.
6. Post-processing refines Markdown through native language/writing adapters with graceful availability checks.
